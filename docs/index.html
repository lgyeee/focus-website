Abstract

Large language models (LLMs) often produce fluent chain-of-thought (CoT) explanations that are unfaithful to the actual decision process, relying on unsupported or hallucinated “facts”. We propose FoCus, a training and inference framework for Condition-Utilized Reasoning. FoCus first extracts explicit conditions from the problem statement and then constrains the model to reason using only cited conditions. Across diverse benchmarks including AIME 2024/2025, MATH-500, and GPQA, FoCus improves faithfulness metrics such as fact utilization and error sensitivity, while largely preserving or improving final-answer accuracy.

Key Contributions

  • We formalize Condition-Utilized Reasoning as a notion of CoT faithfulness: every reasoning step should be grounded in explicit conditions extracted from the input.
  • We introduce a two-stage FoCus pipeline: (1) condition extraction with a format-strict LaTeX template, (2) faithfulness-aware CoT training that encourages explicit citation of conditions.
  • We propose new metrics for leak resistance and error sensitivity, and show FoCus improves these metrics on math and science benchmarks while retaining accuracy.
  • We analyze how FoCus changes the structure of model reasoning, reducing spurious unsupported steps and over-verbose “thinking”.

Method Overview

Two-Stage FoCus Pipeline

  1. Fact / Condition Extraction
    Given a problem, we prompt a model to extract every explicit condition using a strict LaTeX format: \textcolor{red}{<Condition1>} (...), \textcolor{red}{<Condition2>} (...), etc.
  2. Condition-Utilized Reasoning
    We train a reasoning model (or reuse the same one) to solve the problem step-by-step while explicitly citing the relevant conditions for each step.

FoCus can be combined with different base models (e.g., Qwen3-8B, DeepSeek-Qwen3-8B, Nemotron-7B) and evaluation modes (normal, sycophancy-hint, unethical-hint).
